# 우리집 가계부 — 사양서

> 작성일: 2026-05-16
> 프로젝트: 가족 가계부 웹 대시보드
> 주 사용자: 50대 후반 ~ 60대 부모님, 자녀 (가족 4인 기준)

---

## 1. 개요

### 1.1 목적
가족 구성원이 함께 사용하는 가계부 웹 대시보드. 부모님이 주 사용자이지만 자녀도 각자 거래를 입력하고 가족 전체 흐름을 한눈에 볼 수 있어야 한다.

### 1.2 핵심 가치
- **직관성 우선**: 부모님(50~60대)의 시력·조작 패턴 고려, 큰 글씨와 명확한 대비
- **한눈에 파악**: 이번 달 흐름을 첫 화면에서 즉시 이해
- **가족 통합**: 누가 어디에 얼마 쓰는지 투명하게 공유
- **장기 흐름**: 월별·연간 비교로 가계 패턴 파악

### 1.3 비범위 (이번 단계 제외)
- 은행/카드 자동 연동 (API)
- 음성 / OCR 입력
- 모바일 네이티브 앱
- 다국어 (한국어 단일)
- 다중 화폐

## 2. 사용자 & 역할

### 2.1 가족 구성원 모델
| 필드 | 타입 | 설명 |
|---|---|---|
| `id` | string | 고유 식별자 |
| `name` | string | 표시 이름 (예: "아버지", "어머니", "대현", "지원") |
| `short` | string | 1글자 약자 (아바타용) |
| `role` | "가장" \| "자녀" | 가족 내 역할 |
| `color` | string | 아바타 컬러 클래스 |

### 2.2 권한
이번 단계에서는 **모든 가족 구성원이 동일 권한**. 누구든 입력/수정/삭제 가능.
(추후 단계에서 "관리자" 역할 분리 가능성 열어둠)

## 3. 기능 명세

### 3.1 홈 (대시보드)
한 화면에 이번 달 가족 가계 흐름을 보여준다.

**필수 구성 요소**
1. KPI 카드 4종
   - 이번 달 수입
   - 이번 달 지출
   - 남은 예산 (총 예산 - 사용액)
   - 순저축 (수입 - 지출)
   - 각 카드에 전월 대비 증감(%) 부가 정보
2. 이번 달 전체 지출 카드
   - 총액 큰 숫자
   - 카테고리 가로 스택바 (조각 호버/터치 시 툴팁)
   - 카테고리별 타일 그리드 (금액 + 비율)
3. 월별 비교 막대 차트 (최근 6개월)
   - 막대 호버/터치 시 검은 풍선 툴팁 (해당 월의 수입/지출/저축)
4. 카테고리 도넛 차트
   - 조각 호버/터치 시 가운데 큰 숫자가 해당 카테고리로 변경
5. 최근 거래 (5건)
6. 다가오는 결제 (자동이체)
7. 계좌 잔액 요약

**대시보드 변형 (Tweaks로 전환)**
- `card`: 카드 그리드 (기본)
- `big`: 큰 숫자 히어로 (가독성 최우선)
- `family`: 가족 중심 (구성원별 카드 강조)

**가족 구성원 필터 (상단 Pills)**
- 전체 / 아버지 / 어머니 / 대현 / 지원 — 클릭 시 해당 사람 데이터만 표시

### 3.2 거래 내역
필터링 가능한 거래 목록 화면.

**필터**
- 검색 (제목, 메모)
- 수입 / 지출 / 전체
- 카테고리 (다중 또는 단일)
- 가족 구성원
- 기간 (월 단위)

**표시**
- 날짜별 그룹화 (오늘, 어제, 5월 9일 등)
- 각 그룹에 일별 수입/지출 소계
- 거래 한 행 = 카테고리 아이콘 + 제목/메타 + 결제수단 + 금액
- 기간 합계 (수입 / 지출 / 잔액) 상단 노출

### 3.3 거래 입력 (모달)
"+ 새 거래 입력" 버튼으로 열림.

**입력 필드**
| 필드 | 타입 | 필수 | 설명 |
|---|---|---|---|
| 종류 | "지출" \| "수입" | ✓ | 토글 |
| 금액 | number | ✓ | 큰 폰트, 우측 정렬, 천단위 자동 콤마 |
| 카테고리 | string | ✓ | 이모지 타일 4열 그리드 |
| 제목 | string | ✓ | "어디서 / 무엇" |
| 메모 | string | – | 선택 |
| 누가 | member.id | ✓ | 드롭다운 |
| 결제 수단 | account.id | ✓ | 드롭다운 |
| 날짜 | date | ✓ | 기본 오늘 |

**카테고리 (지출 / 수입 별도)**
- 지출: 식비, 교통, 의료/약, 공과금, 여가, 쇼핑, 교육, 기타
- 수입: 월급, 연금, 부수입, 기타

**검증**
- 금액 0 또는 빈 값 차단
- 제목 빈 값 차단
- 미래 날짜 허용 (예약 입력 케이스)

### 3.4 리포트
3개 탭으로 분할된 분석 화면.

#### 3.4.1 가족 통합 탭
- KPI 4종: 가족 전체 수입 / 지출 / 순저축 / 가장 많이 쓴 사람
- 구성원별 카드 4장: 5월 지출 + 비중 + 증감 + 6개월 스파크라인 + 주력 카테고리
- 가족 × 카테고리 매트릭스 (히트맵 컬러 농도)

#### 3.4.2 월별 비교 탭
- 두 달을 드롭다운으로 선택
- KPI 3종 비교: 수입 / 지출 / 순저축 + 증감 %
- 카테고리별 좌우 마주보는 막대 비교
- 가족 구성원별 카드 4장 (두 달 값 + 증감)

#### 3.4.3 연간 정리 탭
- 12개월 KPI: 총 수입 / 총 지출 / 총 저축 + 저축률
- 12개월 라인 차트 (호버 시 따라오는 큰 풍선 툴팁 + 세로 가이드선)
- 베스트/워스트 달 + 월평균 카드
- 카테고리별 1년 트렌드 (스파크라인 + 합계 + 월평균)
- 누적 저축 흐름 라인 차트

### 3.5 예산 · 목표
- 5월 총 예산 헤더 (사용액 / 총 예산 / 사용률 + 진행바)
- 카테고리별 예산 진행률 (8개 카테고리)
  - 100% 초과 시 빨간색
  - 85% 이상 노란색
  - 그 외 카테고리 컬러
- 저축 목표 (적금) 카드형: 제목, 매월 저축액, 누적/목표, 진행률

### 3.6 설정
- 화면 보기
  - 글자 크기 3단: 보통(16px) / 크게(18px) / 아주 크게(20px)
  - 고대비 모드 토글 (테두리·텍스트 진하게)
- 금액 표시 방식 3종
  - `1,234,567원` (기본)
  - `₩1,234,567`
  - `123만 4,567원` (한글 단위)
- 가족 구성원 관리 (이름·역할 수정, 추가)

### 3.7 Tweaks 패널
디자인 검토용 인-페이지 패널. 운영 빌드에서는 비공개 (개발/QA 토글).
- 대시보드 레이아웃 (3종)
- 기본 글자 크기 슬라이더 (15~22px)
- 고대비 모드
- 강조 색상 팔레트 5종
- 금액 표시 방식

## 4. 데이터 모델

### 4.1 거래 (Transaction)
```ts
type Transaction = {
  id: string;
  date: string;          // ISO "YYYY-MM-DD"
  kind: "in" | "out";    // 수입 / 지출
  amount: number;        // 원 단위, 정수
  cat: CategoryId;
  title: string;
  memo?: string;
  member: MemberId;
  account: AccountId;
};
```

### 4.2 계좌 (Account)
```ts
type Account = {
  id: string;
  label: string;         // "국민은행 주거래"
  type: "입출금" | "적금" | "카드" | "현금";
  bank: string;          // "국민", "신한", ...
  balance: number;       // 카드는 음수 (이번 달 사용액)
  color: string;
  limit?: number;        // 카드만 (한도)
};
```

### 4.3 카테고리 (Category)
```ts
type Category = {
  id: "food" | "transport" | "medical" | "utility" |
      "leisure" | "shopping" | "edu" | "other" |
      "salary" | "pension" | "side";
  label: string;
  color: string;         // CSS color
  emoji: string;
};
```

### 4.4 가족 구성원 (Member)
```ts
type Member = {
  id: string;
  name: string;
  short: string;         // 1글자
  role: "가장" | "자녀";
  colorKey: "appa" | "eomma" | "deahyun" | "jiwon"; // 사전 정의 컬러 키
};
```

### 4.5 예산 (Budget)
```ts
type Budget = {
  cat: CategoryId;       // 지출 카테고리만
  limit: number;         // 월 한도
  // 월별 다른 예산이 필요하면 ym(YYYY-MM)로 키 분기
};
```

### 4.6 저축 목표 (Goal)
```ts
type Goal = {
  id: string;
  title: string;
  saved: number;         // 현재 누적
  target: number;        // 목표
  monthly: number;       // 매월 저축액
  color: string;
};
```

### 4.7 고정 지출 / 다가오는 결제 (Upcoming)
```ts
type Upcoming = {
  id: string;
  label: string;
  date: string;          // 결제 예정일
  amount: number;
  cat: CategoryId;
  autopay?: boolean;
};
```

## 5. 화면 흐름

```
[홈]
 ├── (구성원 Pills 클릭) → 홈 (필터 적용)
 ├── (새 거래 입력) → 입력 모달 → 저장 → 홈 갱신
 └── (사이드바 메뉴) → 거래내역 / 리포트 / 예산 / 설정

[거래 내역]
 ├── 검색 / 필터 적용
 ├── (행 클릭) → 거래 상세 모달 (수정·삭제)
 └── (새 거래 입력) → 입력 모달

[리포트]
 ├── 탭: 가족 통합
 ├── 탭: 월별 비교 (두 달 선택)
 └── 탭: 연간 정리

[예산 · 목표]
 ├── (예산 수정) → 카테고리별 한도 편집 모달
 └── (목표 추가) → 저축 목표 추가 모달

[설정]
 ├── 글자 크기 / 고대비 / 금액 표시
 └── 가족 관리 (이름·역할 편집)
```

## 6. 인터랙션 / 상태

### 6.1 차트 툴팁 (필수)
모든 차트에 hover + touch 툴팁:
- 막대 / 도넛 조각 / 라인 포인트 / 스택바 segment
- 검은 배경 풍선 + 흰 텍스트, 12px radius, 부드러운 그림자
- 모바일에서 손가락 떼면 닫힘

### 6.2 키보드 접근성
- 모든 액션 버튼 Tab 이동
- 모달은 Esc로 닫기
- 입력 폼은 Enter로 다음 필드 이동 또는 저장

### 6.3 반응형 분기
- `≥ 1180px`: 사이드바 260px + 메인. KPI 4-up, 본문 2단.
- `880~1180px`: 사이드바 220px + 메인. KPI 2-up, 본문 1단.
- `≤ 880px` (모바일): 사이드바 가로 스크롤 칩, 메인 단일 컬럼.

## 7. 비기능 요구사항

### 7.1 성능
- 첫 페인트(FCP) < 1.5초 (3G 기준)
- 거래 1만건 이상에서도 목록 가상 스크롤 (`react-window` 등)

### 7.2 가독성 / 접근성
- 본문 기본 17px (Tweaks에서 15~22 조정)
- 텍스트 대비 비율 WCAG AA (4.5:1) 이상
- 클릭/터치 영역 최소 44×44px
- 한국어 줄바꿈: `word-break: keep-all` 전역
- 숫자: `font-variant-numeric: tabular-nums` + `white-space: nowrap`

### 7.3 데이터 보존
- 모든 거래는 서버 또는 로컬 영구 저장 (이번 단계는 LocalStorage로 시작 가능)
- 입력 모달 작성 중 새로고침해도 미저장 입력 복원 (자동 임시 저장)

### 7.4 보안 (서버 도입 시)
- 가족 단위로 워크스페이스 분리
- 가족 초대 링크는 24시간 만료
- 거래 데이터는 가족 내 멤버만 조회 가능

## 8. 데이터 보존 / 동기화 (선택)

### 8.1 1단계 (로컬 전용)
- LocalStorage에 가족 워크스페이스 단일 저장

- 거래 입력 시 실시간 동기화
