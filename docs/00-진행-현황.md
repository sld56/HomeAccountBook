### 3️⃣ Cloudflare Tunnel 설정 (15분)

#### 3.1 Tunnel 생성 + DNS 연결
미니 PC PowerShell(관리자):
```powershell
cloudflared tunnel login
cloudflared tunnel create gagyebu
cloudflared tunnel route dns gagyebu bank.hambodev.work
```

또는 Cloudflare Zero Trust 대시보드의 **Public Hostname**에서 만들었다면 이 단계는 이미 끝.

#### 3.2 config.yml 작성
`%USERPROFILE%\.cloudflared\config.yml`:
```yaml
tunnel: gagyebu
credentials-file: C:\Users\sld12\.cloudflared\<UUID>.json
ingress:
  - hostname: bank.hambodev.work
    service: http://localhost:8080
  - service: http_status:404
```
`<UUID>`는 `cloudflared tunnel list`로 확인.

#### 3.3 Windows 서비스로 등록
```powershell
cloudflared service install
Start-Service cloudflared
Get-Service cloudflared
```
Status = Running이면 OK.

### 4️⃣ Resend 메일 설정 (15분)

1. [resend.com](https://resend.com) 가입
2. **Domains → Add Domain** → `hambodev.work` (⚠️ 루트 도메인, `bank.`은 빼고)
3. Resend가 알려주는 DNS 레코드(MX + SPF TXT + DKIM TXT)를 **Cloudflare DNS**에 추가
4. Resend에서 "Verified" 뜰 때까지 대기 (보통 5분)
5. **API Keys → Create API Key** → Permission: Sending → 키 복사 (`re_xxxxx...`)
6. `.env`에 추가:
   ```env
   SMTP_HOST=smtp.resend.com
   SMTP_PORT=587
   SMTP_USER=resend
   SMTP_PASSWORD=re_xxxxxxxxxxxxxxxxxx
   SMTP_ADMIN_EMAIL=noreply@hambodev.work
   SMTP_SENDER_NAME=우리집 가계부
   ```

## 🟧 알려진 잔여 이슈

- **Realtime WebSocket 403** — `/realtime/v1/websocket` 요청이 nginx를 통과 못 함. publication은 정상이지만 nginx/kong WebSocket 프록시 설정 필요. 다른 기능엔 영향 없음 (실시간 동기화만 폴백으로 동작).
- **거래 폼 draft 미저장** — 길게 작성 중 실수로 닫으면 사라짐. 이전 자동 복원이 UX 혼란이라 제거. 추후 명시적 "이어쓰기" 버튼 필요할 수 있음.
- **lib/format.ts dayLabel timezone** — `new Date('YYYY-MM-DD')`가 UTC로 파싱되어 timezone 경계에서 ±1일 오차 가능 (한국은 DST 없어 영향 미미).
- **stale supabase/functions 디렉토리** — `accept-invite/`, `create-invite/` 등 옛 코드 잔존. 라우터는 `main/`이 진실의 원천이지만 추후 정리 필요.
- **studio 컨테이너 unhealthy 23h+** — 영향 없음. supabase/studio:latest 자체 헬스체크 이슈.
