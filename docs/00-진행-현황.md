## 3. 남은 작업 순서 (체크리스트)

각 항목 옆 ⏱️는 예상 소요 시간. 막힐 만한 곳에 💡 힌트.

### 🟦 단계 A. Cloudflare 도메인 활성화 ⏱️ 10분 + 대기 1~24h
- [ ] [cloudflare.com](https://cloudflare.com) 가입 (이미 있으면 skip)
- [ ] **Add a Site** → 본인 도메인 입력 → **Free** 플랜
- [ ] Cloudflare가 알려주는 네임서버 2개를 도메인 등록 업체(가비아·후이즈 등)에서 변경
- [ ] **활성화될 때까지 대기** (Cloudflare 대시보드에서 도메인이 "Active" 표시)
- 💡 이 대기 시간 동안 단계 B를 진행하면 됨

### 🟦 단계 B. 미니 PC 사전 준비 (Windows) ⏱️ 30~40분
**자세한 명령은 [infra/SETUP-WINDOWS.md §1](../infra/SETUP-WINDOWS.md)** 참조

- [ ] BIOS에서 가상화(VT-x/SVM) 활성화
- [ ] 관리자 PowerShell에서 `wsl --install` → 재부팅
- [ ] [Docker Desktop](https://www.docker.com/products/docker-desktop/) 설치 (WSL 2 backend, "Start at sign in" ✅)
- [ ] [Git for Windows](https://git-scm.com/download/win) 설치
- [ ] 자동 로그인 설정 (`control userpasswords2`)
- [ ] 절전 비활성화 (`powercfg /change standby-timeout-ac 0`)
- [ ] Windows Update 활성 시간을 가족 사용 시간대로
- 💡 BIOS 가상화 안 보이면 모델명 알려주세요 — 위치 찾아드림

### 🟦 단계 C. Cloudflare Tunnel ⏱️ 20분
**[infra/SETUP-WINDOWS.md §2](../infra/SETUP-WINDOWS.md)** 참조

- [ ] [cloudflared.exe](https://github.com/cloudflare/cloudflared/releases/latest) 다운로드 → `C:\cloudflared\` 저장 + PATH 추가
- [ ] `cloudflared tunnel login` (브라우저 인증)
- [ ] `cloudflared tunnel create gagyebu`
- [ ] `cloudflared tunnel route dns gagyebu gagyebu.yourdomain.com` ← 단계 A 활성화 후에만 동작
- [ ] `%USERPROFILE%\.cloudflared\config.yml` 작성 (ingress → `http://localhost:8080`)
- [ ] `cloudflared service install` + `Start-Service cloudflared`

### 🟦 단계 D. 프로젝트 가져오기 + 키 생성 ⏱️ 10분
**[infra/SETUP-WINDOWS.md §3](../infra/SETUP-WINDOWS.md)** 참조

- [ ] PowerShell에서 `wsl` 실행해 Ubuntu로 들어가기
- [ ] `sudo apt install -y nodejs git` (Node 22 권장)
- [ ] `git clone https://github.com/sld56/HomeAccountBook.git ~/gagyebu`
- [ ] `cd ~/gagyebu && node infra/scripts/gen-keys.mjs` — 출력 전체 복사
- [ ] `cd infra && cp .env.example .env && chmod 600 .env && nano .env`
- [ ] `.env`에 채우기:
  - `SITE_URL=https://gagyebu.yourdomain.com`
  - `JWT_SECRET`, `POSTGRES_PASSWORD`, `REALTIME_SECRET`, `ANON_KEY`, `SERVICE_ROLE_KEY` ← gen-keys 출력
- [ ] nano: `Ctrl+O`, `Enter`, `Ctrl+X`로 저장 종료
- 💡 Private 레포라서 clone 시 GitHub 인증 필요 — Personal Access Token 또는 Git Credential Manager 안내됨

### 🟦 단계 E. Resend 메일 발송 ⏱️ 15분
**[infra/README.md §3.3](../infra/README.md)** 참조

- [ ] [resend.com](https://resend.com) 가입
- [ ] **Domains** → "Add Domain" → 본인 도메인
- [ ] 표시되는 SPF/DKIM/MX 레코드를 Cloudflare DNS에 추가
- [ ] "Verified" 뜰 때까지 대기 (몇 분)
- [ ] **API Keys** → Create → Sending access → 키 복사
- [ ] `.env`에 추가:
  ```
  SMTP_HOST=smtp.resend.com
  SMTP_PORT=587
  SMTP_USER=resend
  SMTP_PASSWORD=re_xxxxxxxxxxxxxxxxxx
  SMTP_ADMIN_EMAIL=noreply@yourdomain.com
  SMTP_SENDER_NAME=우리집 가계부
  ```

### 🟦 단계 F. Docker 기동 + DB 마이그레이션 ⏱️ 10분

**자세한 절차는 [infra/CI-IMAGES.md](../infra/CI-IMAGES.md) 참조 — 이미지는 GitHub Actions가 미리 빌드해둠.**

- [ ] **GitHub Secrets 등록** (저장소 Settings → Secrets and variables → Actions)
  - `VITE_SUPABASE_URL` = SITE_URL과 동일
  - `VITE_SUPABASE_ANON_KEY` = gen-keys의 ANON_KEY
- [ ] **Actions 첫 빌드 트리거** — GitHub Actions 탭 → "Build and Publish Docker Image" → Run workflow → 5~7분 대기
- [ ] **미니 PC에서 ghcr.io 로그인** (PAT 발급 후)
  ```bash
  echo "<PAT>" | docker login ghcr.io -u sld56 --password-stdin
  ```
- [ ] `cd ~/gagyebu/infra && docker compose up -d` (--build 불필요, pull만)
- [ ] `docker compose ps` — 모든 컨테이너 healthy 확인
- [ ] DB 스키마 적용:
  ```bash
  docker compose exec -T postgres psql -U postgres -d postgres < ../supabase/migrations/0001_init.sql
  docker compose exec -T postgres psql -U postgres -d postgres < ../supabase/migrations/0002_rls.sql
  ```

**이후 업데이트**: `git push` (본인 PC) → Actions 자동 빌드 → 미니 PC에서 `docker compose pull && up -d` (1분)

### 🟦 단계 G. 첫 접속 + 가족 가입 ⏱️ 15분
- [ ] 브라우저로 `https://gagyebu.yourdomain.com` 접속
- [ ] 본인 이메일로 로그인 → 6자리 코드 입력
- [ ] "새 가족 만들기" → 가족 이름, 표시 이름, 색상 선택
- [ ] (선택) "기존 데이터 가져오기" 모달이 뜨면 마이그레이션
- [ ] **설정 → 가족 구성원 (서버)** → 가족 이메일로 초대 발송
- [ ] 가족이 메일 링크 클릭 → 가입 → 합류

### 🟦 단계 H. 백업 자동화 ⏱️ 15분
**[infra/SETUP-WINDOWS.md §7](../infra/SETUP-WINDOWS.md)** 참조

- [ ] [age.exe](https://github.com/FiloSottile/age/releases) 설치
- [ ] `age-keygen -o C:\backup-keys\gagyebu.key` (공개키 메모)
- [ ] `.env`에 `BACKUP_DIR`, `BACKUP_AGE_RECIPIENT` 추가
- [ ] `.\infra\scripts\backup.ps1` 수동 실행해서 백업 파일 생성 확인
- [ ] Task Scheduler에 매일 03:00 등록
- ⚠️ **age 비밀키를 USB나 클라우드 드라이브에도 별도 보관** — 잃어버리면 복호화 불가

### 🟦 단계 I. 모니터링 (선택) ⏱️ 10분
- [ ] Uptime Kuma 컨테이너 추가
- [ ] HTTP 모니터로 `https://gagyebu.yourdomain.com` 등록
- [ ] 카카오톡 또는 이메일 알림 설정

## 5. 자주 쓰는 명령어 모음

### WSL Ubuntu 안 (배포 작업용)
```bash
cd ~/gagyebu

# 코드 업데이트 받기
git pull
docker compose -f infra/docker-compose.yml up -d --build

# 서비스 상태
docker compose -f infra/docker-compose.yml ps
docker compose -f infra/docker-compose.yml logs -f gotrue   # auth 로그

# 재시작
docker compose -f infra/docker-compose.yml restart gotrue

# 백업 수동 실행
ENV_FILE=~/gagyebu/infra/.env /usr/local/bin/gagyebu-backup
```

### Windows PowerShell (관리자)
```powershell
# Cloudflare Tunnel 상태
Get-Service cloudflared
Start-Service cloudflared
Restart-Service cloudflared

# 백업 수동 실행
.\infra\scripts\backup.ps1

# Task Scheduler 백업 작업 상태
Get-ScheduledTask -TaskName "GagyebuBackup" | Get-ScheduledTaskInfo
```

### 로컬 개발(Windows의 본인 PC)
```powershell
# 의존성
npm install

# 개발 서버
npm run dev

# 타입 체크 + 빌드 검증
npm run typecheck
npm run build
```
